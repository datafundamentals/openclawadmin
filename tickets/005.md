# Ticket 005 — Persist Auditor Outputs and Final Reports

## Title

Persist auditor section outputs and final SecurityCouncil reports to `vm_reports/`

---

# Problem

Currently the outputs of the LLM auditors and the report aggregator exist only in:

* terminal output
* chat session history
* manual copy/paste

This introduces several risks:

* loss of audit records
* inability to reproduce previous reports
* no historical audit trail
* fragile handoff between pipeline stages

The system should instead persist all outputs to disk.

---

# Goal

Modify the audit workflow so that:

1. each auditor writes its output to a file
2. the report aggregator reads those files
3. the final report is written to `vm_reports/`
4. each audit run produces a permanent record

This removes dependence on terminal history and makes the system **fully reproducible**.

---

# Scope

Introduce structured report output directories under:

```text
vm_reports/
```

Each run should produce its own report directory.

---

# Directory Structure

Each audit run should create a directory:

```text
vm_reports/<run_id>/
```

Example:

```text
vm_reports/2026-03-06T19-45-14Z/
```

Inside the run directory:

```text
vm_reports/<run_id>/
  sections/
    configuration.md
    dependency.md
    cost_usage.md
    exposure.md
  security-council-report.md
  run-metadata.json
```

---

# Run Metadata

Create a small metadata file describing the audit run.

File:

```text
vm_reports/<run_id>/run-metadata.json
```

Example structure:

```json
{
  "run_id": "2026-03-06T19-45-14Z",
  "artifact_timestamp": "2026-03-06T19:45:14Z",
  "collector_version": "0.1",
  "execution_mode": "read-only",
  "auditors": [
    "configuration",
    "dependency",
    "cost_usage",
    "exposure"
  ]
}
```

The `artifact_timestamp` should come from:

```text
vm_artifacts/metadata.json
```

---

# Auditor Output Persistence

Each auditor should write its output to:

```text
vm_reports/<run_id>/sections/
```

Files:

```text
configuration.md
dependency.md
cost_usage.md
exposure.md
```

These files should contain **exactly the section output produced by the auditor**.

Example:

```text
vm_reports/2026-03-06T19-45-14Z/sections/configuration.md
```

---

# Report Aggregator Input

The report aggregator must read from these files instead of terminal text.

Inputs:

```text
vm_reports/<run_id>/sections/configuration.md
vm_reports/<run_id>/sections/dependency.md
vm_reports/<run_id>/sections/cost_usage.md
vm_reports/<run_id>/sections/exposure.md
vm_reports/<run_id>/run-metadata.json
```

---

# Final Report Output

The final SecurityCouncil report must be written to:

```text
vm_reports/<run_id>/security-council-report.md
```

This file becomes the **authoritative audit record**.

---

# Script Behavior

Any orchestration script (for example `audit-run.sh`) should:

1. create the run directory
2. run auditors
3. store section outputs
4. run the aggregator
5. write the final report

Example output message:

```text
Audit completed
Report written to vm_reports/2026-03-06T19-45-14Z/security-council-report.md
```

---

# Design Constraints

The persistence system must:

* never overwrite previous reports
* create a new directory for each run
* keep section outputs immutable after creation
* maintain clear separation between sections and final report

This ensures audit runs are **historically traceable**.

---

# Acceptance Criteria

Running the audit pipeline should produce a directory like:

```text
vm_reports/2026-03-06T19-45-14Z/
```

Containing:

```text
sections/configuration.md
sections/dependency.md
sections/cost_usage.md
sections/exposure.md
security-council-report.md
run-metadata.json
```

The final report must match the schema in:

```text
schemas/security-council-report.md
```

---

# Out of Scope

This ticket does not:

* modify artifact collection
* modify baseline logic
* modify drift detection
* modify auditor reasoning

It only persists outputs to disk.

---

# Result

After this ticket is implemented:

* audit runs become **fully reproducible**
* reports become **permanent artifacts**
* auditors become **file-driven**

The SecurityCouncil pipeline becomes:

```text
 ↓
sync
 ↓

 ↓

aggregator

vm_reports/<run_id>/
```
