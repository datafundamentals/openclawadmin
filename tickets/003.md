# Ticket 003 — Implement Artifact Validation Tool

## Title

Create `validate-artifacts.sh` to verify artifact integrity before audits run

---

# Problem

The SecurityCouncil audit pipeline assumes that artifacts collected from the VM are:

* present
* correctly formatted
* structurally valid

Currently there is **no validation step** between artifact collection and auditor execution.

If artifacts are:

* missing
* truncated
* malformed
* partially written
* schema-incompatible

then auditors may produce incorrect reports or misleading findings.

---

# Goal

Introduce a deterministic artifact validation step that verifies the artifact set before auditors run.

The validation tool should ensure that:

1. all required artifact files exist
2. JSON artifacts are valid JSON
3. required keys are present
4. artifact files are non-empty
5. artifact structure matches the artifact contract

If validation fails, the audit pipeline should **stop immediately**.

---

# Scope

Create a new script:

```id="0xgbyi"
scripts/validate-artifacts.sh
```

This script will validate artifacts located in:

```id="hnsf26"
vm_artifacts/
```

---

# Required Behavior

The validation script must perform the following checks.

---

# Check 1 — Required Files Exist

Verify the presence of all expected artifacts.

Required files:

```id="4c6j21"
vm_artifacts/config.json
vm_artifacts/metadata.json
vm_artifacts/packages.txt
vm_artifacts/ports.txt
vm_artifacts/firewall.txt
vm_artifacts/usage.json
```

If any file is missing, the script must exit with an error.

Example failure message:

```id="kq8hrh"
ERROR: Missing artifact file vm_artifacts/config.json
```

---

# Check 2 — Files Are Not Empty

Verify each artifact file is non-empty.

Example check:

```id="d3c76p"
if [[ ! -s vm_artifacts/config.json ]]; then
  echo "ERROR: config.json is empty"
  exit 1
fi
```

---

# Check 3 — JSON Syntax Validation

Validate JSON artifacts using `jq`.

Artifacts requiring validation:

```id="o6qp9j"
config.json
metadata.json
usage.json
```

Example check:

```id="d5b23g"
jq empty vm_artifacts/config.json
```

If parsing fails:

```id="p3gx6h"
ERROR: config.json is not valid JSON
```

---

# Check 4 — Required JSON Keys

Ensure required keys exist.

## metadata.json

Required keys:

```id="cxdekn"
generated_at_utc
collector_version
```

Example check:

```id="dszyo5"
jq -e '.generated_at_utc' vm_artifacts/metadata.json > /dev/null
```

---

## config.json

Required keys:

```id="xpbfa6"
browser_tool_enabled
shell_tool_enabled
plugin_system_enabled
ollama_access_enabled
approved_llm_providers
```

---

## usage.json

Required keys:

```id="8h9qk7"
anthropic_calls
openai_calls
gemini_calls
cap_threshold_warning
```

---

# Check 5 — packages.txt Structure

Verify that the file appears to be `dpkg -l` output.

Example heuristic check:

```id="m9f83q"
grep -q "^ii" vm_artifacts/packages.txt
```

If no installed packages appear:

```id="4ljp9r"
ERROR: packages.txt does not contain expected dpkg entries
```

---

# Check 6 — ports.txt Structure

Verify the file contains LISTEN entries.

Example:

```id="w4vy67"
grep -q "LISTEN" vm_artifacts/ports.txt
```

If none are found:

```id="m3gktb"
ERROR: ports.txt contains no LISTEN entries
```

---

# Check 7 — firewall.txt Structure

Verify that firewall output contains expected markers.

Example:

```id="o0d5yh"
grep -q "Status:" vm_artifacts/firewall.txt
```

---

# Script Behavior

The script must:

* fail fast
* print clear diagnostic errors
* exit with non-zero status on failure

Include:

```id="of2dkw"
set -euo pipefail
```

If all checks pass, print:

```id="gq5crn"
Artifact validation successful
```

---

# Integration With Audit Pipeline

This script will be called by the future pipeline orchestrator.

Example:

```id="q8d6hx"
scripts/validate-artifacts.sh
```

The pipeline must stop if validation fails.

# Acceptance Criteria

Running:


should:

* verify artifact presence
* verify JSON validity
* verify required keys
* verify expected text structure

If validation passes:

