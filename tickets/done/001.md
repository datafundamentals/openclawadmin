# Ticket 001 — Normalize Collector Artifacts for Deterministic Audits

**Status: ✅ DONE**

## Title

Normalize collector artifacts to remove runtime-volatile fields

## Problem

Current artifact collection includes runtime-volatile fields that change every run, causing unnecessary diffs between `vm_artifacts` and `vm_baseline`.

Examples:

* `timestamp_utc`
* `generated_at_utc`
* process IDs in `ports.txt`
* file descriptors in `ports.txt`

These fields represent **runtime state**, not configuration drift, and therefore should not cause baseline comparison noise.

Example current diff noise:

```
pid=899 vs pid=938
fd=3 vs fd=4
timestamp_utc changed
```

This makes artifact comparisons less deterministic.

## Goal

Modify the collector so that artifacts represent **semantic system state** rather than runtime identifiers.

Artifacts should remain stable across repeated runs unless **actual system configuration changes** occur.

## Scope

Modify:

```
collector/collect.sh
```

## Required Changes

### 1. Normalize `ports.txt`

Current collection uses:

```
ss -lntp
```

This includes runtime values:

```
pid=<number>
fd=<number>
```

Modify the collector to normalize these values.

Replace with:

```
sudo ss -lntp \
  | sed -E 's/pid=[0-9]+/pid=PID/g' \
  | sed -E 's/fd=[0-9]+/fd=FD/g' \
  | sort \
  > artifacts/ports.txt
```

Result example:

```
users:(("sshd",pid=PID,fd=FD))
```

This preserves service identity while removing runtime volatility.

### 2. Ensure deterministic ordering

Add `sort` to the pipeline to prevent ordering changes between runs.

### 3. Timestamp handling

Artifact timestamps should exist only in:

```
metadata.json
```

Ensure that:

* `config.json`
* `usage.json`

do **not include timestamps** if they currently do.

Timestamps should remain only in:

```
artifacts/metadata.json
```

Example:

```json
{
  "generated_at_utc": "2026-03-06T19:45:14Z",
  "collector_version": "0.1"
}
```

Auditors should treat metadata timestamps as informational and ignore them for drift detection.

## Acceptance Criteria

Running the collector twice in a row should produce artifacts where:

```
diff vm_artifacts vm_baseline/artifacts.baseline
```

shows **no changes** except when actual system state changes.

Specifically:

* no PID differences
* no FD differences
* stable port listing order

## Out of Scope

Do not modify:

* auditor prompts
* report schemas
* baseline comparison logic

This ticket is limited to **collector artifact normalization**.
