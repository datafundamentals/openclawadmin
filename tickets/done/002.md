# Ticket 002 — Implement Single-Command Audit Pipeline

**Status: ✅ DONE**

**Implementation note:** The original ticket specified a bash script. After evaluation,
the implementation was replaced with `scripts/audit-run.py` (Python + LiteLLM) for
reliable LLM invocation with configurable model routing. The bash version (`audit-run.sh`)
was deleted as part of ticket 006. LLM and model configuration lives in `.env` (gitignored);
see `.env.example` for the template.

## Title

Create `audit-run.sh` to execute the full SecurityCouncil audit pipeline

## Problem

The current audit workflow requires several manual steps:

1. Run collector on VM
2. Sync artifacts to host
3. Run auditors
4. Run report aggregator
5. Store report output

This workflow is functional but cumbersome and error-prone.

## Goal

Provide a single command that executes the full audit pipeline.

Example:

```
./audit-run.sh
```

This should orchestrate the complete process.

## Scope

Create a new script:

```
scripts/audit-run.sh
```

## Required Behavior

The script should perform the following steps in order.

### Step 1 — Trigger collector on VM

Execute the collector remotely.

Example pattern:

```
ssh <vm> "cd ~/openclawadmin && ./collector/collect.sh"
```

Use the same VM configuration already used by:

```
scripts/sync-from-vm.sh
```

Do not duplicate connection configuration unnecessarily.

### Step 2 — Pull artifacts

Run the existing sync script:

```
scripts/sync-from-vm.sh
```

Artifacts should populate:

```
vm_artifacts/
```

### Step 3 — Run auditors

Execute the four auditors using the prompts in:

```
prompts/
```

Auditors:

```
configuration-auditor
dependency-auditor
cost-usage-auditor
exposure-auditor
```

Each auditor should produce a section output.

Temporary section outputs should be written to:

```
vm_reports/sections/
```

Example:

```
vm_reports/sections/configuration.md
vm_reports/sections/dependency.md
vm_reports/sections/cost_usage.md
vm_reports/sections/exposure.md
```

### Step 4 — Run report aggregator

Use:

```
prompts/report-aggregator.txt
```

Aggregator should synthesize the four auditor outputs and produce the final report.

Output file:

```
vm_reports/security-council-report-<timestamp>.md
```

### Step 5 — Print summary

The script should print:

```
Audit completed
Report written to vm_reports/<filename>
```

## Design Constraints

The script must:

* remain **read-only with respect to the VM**
* never modify system configuration
* only collect and analyze artifacts

The script should fail fast on errors:

```
set -euo pipefail
```

## Acceptance Criteria

Running:

```
./scripts/audit-run.sh
```

should:

1. collect artifacts
2. sync them
3. run auditors
4. produce a report

with no manual steps required.

Final report should appear in:

```
vm_reports/
```

## Out of Scope

Do not modify:

* collector behavior
* artifact schema
* baseline logic

This ticket only introduces orchestration.
