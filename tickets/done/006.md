# Ticket 006 — Retire Bash Audit Pipeline

**Status: ✅ DONE** — `scripts/audit-run.sh` deleted after `scripts/audit-run.py` validated end-to-end.

## Title

Remove `scripts/audit-run.sh` once Python pipeline is validated

## Problem

Two parallel paths exist for running the audit pipeline:

```
scripts/audit-run.sh    (bash, ticket 002 — unreliable LLM invocation)
scripts/audit-run.py    (Python, ticket 002 revised — LiteLLM, configurable)
```

Maintaining both creates ambiguity about which is authoritative and increases
the risk of human error (running the wrong one, making changes to only one, etc.).

## Goal

Once `scripts/audit-run.py` has been validated through at least one successful
end-to-end audit run, retire the bash version.

## Required Changes

1. Delete `scripts/audit-run.sh`
2. Update any documentation that references `audit-run.sh`
3. Verify no other scripts call or depend on `audit-run.sh`

## Acceptance Criteria

- `scripts/audit-run.sh` no longer exists
- `scripts/audit-run.py` is the single authoritative entry point for audit runs
- `README.md` or equivalent reflects the Python invocation

## Prerequisite

- Ticket 002 (Python version) validated through at least one successful run

## Out of Scope

- Changes to collector, sync scripts, or prompts
- Changes to artifact schema or baseline logic
